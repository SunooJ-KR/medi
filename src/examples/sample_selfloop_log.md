# 자가 루프 검증 — 생성→검증→재생성 100% PASS

> 작업 4-3 산출물. AI가 생성한 카피가 `validate_copy.py`를 100% 통과할 때까지 재생성되는지 확인.
> 핵심 구조: **생성(AI) → 검증(코드) → 위반 시 재생성(AI) → 재검증**, 최대 3회. 미해결 시 🔴 강등.

## 시나리오

- 시장: 일본(JP), 룰셋 `rules/jp.json` v1.1.0
- 페르소나: 30대 칙칙함/건조 (copy_direction: 우루오이·투명감 예방/정돈 범위, 미백·시미제거 금지)
- 제품 효능: 비타민C 브라이트닝

## Iteration 1 — 1차 생성 카피 (검증 실패)

AI 1차 생성:
```
ビタミンCで美白効果、シミを消すように明るい肌へ。
```

검증(엔진 실제 출력):
```
verdict   VIOLATION
pass_rate 0/1 문장 통과
JP-002  美白        VIOLATION
JP-001  シミを消す   VIOLATION
```

미해결 VIOLATION 존재 → 재생성 트리거. alternatives 참조:
- JP-002 → `透明感のある印象へ` / `メーキャップ効果で明るい肌印象`
- JP-001 → `日焼けによるシミ・そばかすを防ぐ`

## Iteration 2 — 재생성 카피 (검증 통과)

AI 재생성:
```
ビタミンC配合、うるおいで透明感のある明るい印象へ。日焼けによるシミ・そばかすを防ぐ。
```

검증(엔진 실제 출력):
```
verdict   PASS
pass_rate 2/2 문장 통과
```

## 결과

| 반복 | verdict | 통과율 | 조치 |
|------|---------|--------|------|
| 1 | 🔴 VIOLATION | 0/1 | alternatives 참조 재생성 |
| 2 | 🟢 PASS | 2/2 | 승인 확정 |

- 2회 내 100% PASS 달성. 루프 상한(3회) 미도달.
- **판정은 전적으로 코드**: AI는 생성·재생성만 담당, 통과 여부는 `validate_copy.py`가 결정 → 규제 판정 환각 원천 차단.
