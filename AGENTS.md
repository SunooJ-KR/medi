# Repository Instructions

## Codex Progress Tracking

- Before starting any task, read `.codex/progress.md`.
- Review the current task, pending issues, next recommended step, and recent changes before working.
- Before ending a task, update `.codex/progress.md`.
- Keep entries concise, fact-based, and written in Korean.
- Every entry must include changed files, tests or checks run, remaining issues, and the next step.
